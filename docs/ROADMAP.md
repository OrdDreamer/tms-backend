# Roadmap

## Legend

| Priority | Description |
|----------|-------------|
| 🔴 Must Have | Critical for production readiness |
| 🟠 Should Have | Important for core functionality |
| 🟡 Could Have | Valuable but not urgent |
| 🔵 Nice to Have | Long-term or low-priority improvements |

---

## 🔴 Must Have

- [ ] **Access control** — roles and permissions system
- [ ] **Search** — across projects, translation keys, and translations

---

## 🟠 Should Have

- [ ] **Data import** — import translations from external files/sources
- [ ] **Translation automation** — machine translation and AI-assisted suggestions

---

## 🟡 Could Have

- [ ] **Custom languages** — ability to add a custom language to a project
- [ ] **Export access control** — token-based or public/private project visibility
- [ ] **Translation drafts** — save translations as drafts before publishing
- [ ] **Comments & suggestions** — per-translation discussion and proposals
- [ ] **Project analytics** — usage and activity statistics
- [ ] **Quality control** — duplicate detection, error checking, consistency validation
- [ ] **Glossary templates** — predefined sets of keys and translations reusable across projects
- [ ] **Notifications & reports** — alerts for project changes, reviews, and comments
- [ ] **Content versioning** — history of translation changes with rollback support

---

## 🔵 Nice to Have

- [ ] **GitLab integration** — detect newly created keys and automate their creation
- [ ] **App integrations** — connect with external applications for usage analytics
